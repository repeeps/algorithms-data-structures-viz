# 알고리즘·자료구조, 눈으로 보기 (Algorithms & Data Structures, Visualized)

정렬·탐색·그래프·DP가 **한 단계씩 어떻게 풀리는지** 코드 없이 직접 ▶재생하며 익히는 인터랙티브 학습 자료입니다. 모든 과정이 브라우저에서 실제 로직으로 계산·기록·재생됩니다.

> 직접 만든 **비영리 학습용 자료**입니다. 특정 교재·강의와 무관하며, 설명·예시·코드는 모두 독자적으로 제작했습니다.

| 02 · 과정을 재생한다 | 버블 정렬을 ▶재생·⏭한 단계로 — 이 트랙의 핵심 도구 |
| 03 · 코스 지도 | 19개 챕터 흐름 + 추천 순서 |

## 새로운 조작법 — 스텝 플레이어

다른 트랙이 슬라이더로 "값"을 보았다면, 여기선 **시간축을 따라 과정을 봅니다**: `▶재생 · ⏸ · ⏮ · ⏭한 단계 · 스크럽 · 속도(0.5×/1×/2×)`. **"다음에 뭐가 일어날까?" 예측하고 ⏭로 확인**하는 능동 학습이 핵심입니다.

## 시작하기

빌드·의존성 없음. `index.html` 을 브라우저로 열면 끝입니다.

```bash
open index.html                 # macOS
python3 -m http.server 8000     # → http://localhost:8000
```

## 구조

```
index.html          목차 허브
shared/theme.css    공통 디자인 시스템 (다크 테마) + 스텝 플레이어/범례 스타일
shared/lib.js       공통 헬퍼 + 알고리즘 엔진(VZ.AL) + 차트(VZ.linePlot)
NN-주제.html         챕터별 단일 파일 (각자 독립 실행)
```

### VZ.AL 엔진 — "기록-재생"

알고리즘을 한 번 돌리며 상태 스냅샷을 쌓고(`recorder`), 그 프레임들을 플레이어가 재생합니다(`player`·`transport`). 그 위에 자료구조 렌더러를 얹습니다: `bars`(정렬 막대) · `grid`(격자/미로) · `dpTable`(2D DP 표) · `graph`(노드·엣지) · `tree`(재귀 호출/이진트리) · `legend`(상태 색 범례).

## 설계 원칙

- **실시간 계산**: 정렬·탐색·BFS/DFS·다익스트라·DP가 모두 실제 코드로 동작 (하드코딩 결과 없음)
- **시청이 아닌 능동 학습**: 예측→재생→확인, 작은 입력 손으로 따라가기, 나란히 비교, 상태 패널 동기화
- **코드·수식 없는 직관 우선**: Big-O는 "입력 2배면 일 몇 배?"로, 용어는 등장할 때마다 풀이
- **단계별 분해**: 각 챕터 4스텝, 직관 → 한 단계씩 → 크게/비교 → 정리(+실무 연결)
